Why choose a UI framework ?

Why is it so difficult to choose ?

So many JS frameworks

The reasons

  • Web browser evolution
  • the entirety of your app framework must be downloaded and parsed before users even see your app

More detail...

Points

  • Full MVC framework ? Only the View layer ?
  • Which features do we need? 2-way binding, routing ?
  • Simple to learn, easy to read ?
  • Library size (fast to download ?)
  • Easy to integrate in existing applications ?
  • IE8 compatibility ?
  • Good support from the community

The contenders

  • AngularJS
  • React
  • Riot

AngularJS

http://angularjs.org

35756*

Concept

AngularJS is a full-stack framework with controllers, services, custom tags (called "directives")

Templates are regular html with special AngularJS attributes ng-*=

  • ng-repeat
  • ng-show
  • ng-class
  • ...

Pros

  • Full MVC framework
  • The power of "dependency injection" (unit test)
  • Templates easy to write
  • A lot of magic
  • Big community
  • A lot of exciting projects!

Cons

  • Does not play well with legacy (jQuery plugins)
  • You have the learn the "Angular way"
  • Building custom components (using "directives”) is not very convenient
  • Not compatible with IE8 since v1.3
  • Too much magic!

The future

A lot of exciting projects!

  • Material Angular
  • Ionic(mobile hybrid framework)

Angular 2.0 will change everything!

React

http://facebook.github.io/react/

17198*

Concept

Create re-usable components using JSX (a mix of html and JS, but it is mainly JavaScript, see below)

Reacts promotes "Explicitness and modularity".

Pros

  • Modular approach, everything is a component
  • Focus on performance, fast in the browser
  • IE8 compatible

Cons

  • JSX templates are not very user friendly, a "very JavasScript" approach.
    • No "for each” helper, you do loop calling the map() function on an array
    • No "show" / "hide" helper
    • You use "className" instead of “class”
  • Only the View layer

The future

The project is growing fast

Several interesting projects use React: reapp.io for example (hybrid app framework)

Riot

https://muut.com/riotjs

5531*

Concept

Create "custom html tags", that contain both the view and the business logic.

Riot compilation can be done in the browser.

API is very simple, a few view helpers are provided, but it all that we need!

  • each
  • show
  • hide
  • class

Pros

  • Modular approach with "custom tags"
  • Easy to learn, a simple API
  • Easy to read: template (markup) and JavaScript code in the same place.
  • A super light library (fast to download)
  • IE8 compatible

Cons

  • Very young project (Riot 2.O was released in 2015 January), some bugs, small community
  • Only the View layer
